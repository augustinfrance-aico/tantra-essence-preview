2 · The Kajabi reality

Three paths, plain English

Based on your message and the AI breakdown, there are 3 ways to put custom design into Kajabi:

★ Recommended

Path A — Custom Code Blocks

Kajabi pages have a "Custom Code" block where you can paste raw HTML, CSS, and JavaScript. Our entire GitHub Pages design pastes directly into Kajabi page sections.

Pros: fast, no Liquid required, zero adaptation needed (our code is vanilla) Cons: limited to what fits in Custom Code blocks (one per section), can't edit the global wrapper

Path B — Theme Editing in Liquid

Higher Kajabi plans let us edit the .liquid templates of the theme directly (the path you showed in your Loom).

Pros: full control over every layout Cons: requires Liquid template knowledge, more complex, riskier

Path C — External hosting + iframe

Host on Vercel/Netlify, embed in Kajabi via iframe or subdomain.

Pros: total freedom on the external page Cons: loses native Kajabi forms, opt-ins, and offer purchases — would force us to rewire everything via API

My recommendation: Path A first, Path B only if needed

Our existing GitHub Pages design is vanilla HTML/CSS/JS — exactly what Custom Code blocks expect. Zero adaptation. We keep all native Kajabi integrations (forms, opt-ins, checkouts), we iterate fast, and we never lose Sandra's data.

Path B only if we hit a wall Path A can't reach. Path C avoided.
